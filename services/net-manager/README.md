# net-manager (optional, not active)

> **Status: NOT USED in current deployment.**
> Peer configs are generated by `scripts/entrypoint-simple.sh` (Docker) or `scripts/vpn-gateway-setup` (native).

## What is this?

Python service for automatic UPnP port forwarding and WireGuard peer config generation.
Designed for running VPN Gateway **behind a home router** (not on a VPS).

## What it does

- Discovers UPnP-enabled router via SSDP
- Automatically forwards WireGuard port (51820/UDP) through the router
- Detects external IP via UPnP `GetExternalIPAddress()`
- Generates WireGuard peer configs with the correct public endpoint
- Serves configs via HTTP on port 8088

## Why it's not used

The current deployment model is **single-container on a VPS** with `network_mode: host`.
On a VPS there is no router, so UPnP is irrelevant. The server IP is known in advance
and set via `WG_SERVER_URL` environment variable.

## When to use

If you want to run VPN Gateway at home behind a NAT router, this service can
automate port forwarding. It would need to be added back to `docker-compose.yml`
as a separate service with macvlan networking for LAN/UPnP access.
